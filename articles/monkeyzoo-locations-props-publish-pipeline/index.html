MonkeyZoo Workflow: Locations and Props Get a Home, and the Publish Pipeline Goes Live

Originally shared by on X · retrieved 2026-07-15. This is added context and credit, not the original work.

I have been building the MonkeyZoo Production Workflow System to solve a very specific problem: keeping a comicbook series consistent across seasons, artists, and a lot of moving pieces. Locations, props, characters, art prompts, published issues, all of it needs one place to live and one set of rules for how it moves from idea to finished page. That is what this repo is.

What just shipped

The last day of work landed on two fronts.

First, canon content. I added primary-reference images for every approved location and prop, and locked in the season locations and props for the Aug through Jan "Signal" story arc. That gives the series an actual visual bible to draw from instead of scattered notes.

Second, the tooling to use that canon. I built foundation browsers in the UI so locations and props can actually be looked through and referenced, plus a Studio control panel for the art prompt pack and for publishing archives. Underneath that, I built out the ops side: an art prompt pack workspace, archive publishing, and backups, along with a promote/replace flow that keeps archives unique and leaves live test evidence behind so I can trust a publish actually worked.

There is also a real-issue published run now, with an operator startup flow behind it, plus a batch of stability fixes: static web assets locked to LF line endings so cache-busting works correctly, and static safety and character resolution stabilized so the system does not choke on edge cases. I also wrote up a Phase 0 local ship baseline doc so there is a clear line in the sand for what "working locally" means before anything moves further downstream.

From canon reference to a published issue: the shape of the new pipeline.

Why it matters

A comic series lives or dies on consistency. If a location or a prop drifts between issues because nobody remembers what it looked like three months ago, readers notice. Having a canon library with real reference images, browsable in the UI, means anyone working on an issue (including future me) can pull up the actual approved version of a place or object instead of guessing. And the publish pipeline work means getting an issue from draft to shipped is turning into a real, repeatable process instead of a one-off scramble.

Rick's take

This is the part of building MonkeyZoo I like most: the boring infrastructure that makes the fun part possible. Locations and props sound small, but they are the backbone of a visual story, and getting them into a real browsable system with reference art attached is exactly the kind of unglamorous work that pays off every single issue after this one. The publish/archive pipeline landing in the same window feels good too. Promote-replace with unique archives and live test evidence is the difference between "I think it published" and "I know it published," and that matters a lot once there is a real audience reading these issues.

What's next

With the Phase 0 local ship baseline documented and a real issue already published through the new pipeline, the next stretch is about building on that foundation: more canon content for the Signal arc, and continuing to round out the Studio tooling around it. I will keep logging it here as it moves.

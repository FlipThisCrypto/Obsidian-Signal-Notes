MonkeyZoo Production Workflow: Round 2 Ships Foundation-Level Hardening

Originally shared by on X · retrieved 2026-07-22. This is added context and credit, not the original work.

I have been building out the production workflow system for MonkeyZoo, a comic book series I am developing, and the last 24 hours were a full engineering sprint on making the whole thing genuinely reliable. Here is where things stand.

What just shipped

Round 2: infrastructure integrity

This was the main push. I added a CSRF trust boundary to the writable Studio service so the web interface is no longer open to cross-site mutations. The continuity ledger, which is the canon source of truth for the series, now has an integrity gate so nothing quietly corrupts the storyline data. Per-character bible edits are serialized so concurrent edits cannot stomp each other. I wired in verifiable backups for disaster recovery. A readiness probe now covers all critical data roots, a structured operations log tracks every mutation for observability, and the issue list is verified to match what is actually operable, with no ghost issues slipping through.

Art pipeline: resilience and failure reporting

The art generation batch runner now fails cleanly when a prompt pack is missing or malformed. It survives a ComfyUI outage and reports which submissions failed rather than pretending everything went fine. Failed runs exit non-zero so the calling pipeline knows something went wrong.

Canon validation and testing

Script validation and the experimental-trait advisory now use whole-word character matching, so a character named Bo does not accidentally flag Bones. The find_art function is now deterministic with initial tests in place. Regression tests cover the orchestrator's canon-integrity gates.

Tooling polish

The new_issue CLI now deprecates cleanly with a real message instead of crashing on bad arguments. Character portrait export compares file content rather than just file size, which catches subtle regeneration mismatches a size check would miss.

Every mutation goes through the gate before it touches the canon.

Why it matters

Comic production is not just a creative problem. It is a data integrity problem. Character bibles, location references, prop galleries, and issue scaffolding all need to stay in sync, or continuity breaks down fast. The work this round is about making the workflow trustworthy before the volume of issues and characters gets large enough to hide errors.

Rick's take

I am genuinely enjoying the discipline of treating this like a production system rather than a hobby project. The CSRF work, the serialized edits, the readiness probes: none of it shows up in the finished comic. But it is the reason the comic will actually get made without spending half my time untangling data corruption. Every clean iteration gives me more confidence to push the creative side harder. That trade-off feels right.

What's next

The readiness probe and operations log being in place means the system now has a voice. Next is expanding what it reports and pushing more art pipeline coverage. The foundation is solid enough to start stacking real creative workflow on top of it.

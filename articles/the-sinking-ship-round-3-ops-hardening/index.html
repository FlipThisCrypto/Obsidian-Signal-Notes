The Sinking Ship: Round 3 Done, Chest Verifier Live, Mint Ops Built to Last

Originally shared by on X · retrieved 2026-07-22. This is added context and credit, not the original work.

The Sinking Ship is my Chia NFT collection project. It is built around chests, grails, and characters from the Bepe universe, and it has been in heavy development with one clear goal: get the mint infrastructure right before anything goes public. This sprint was the biggest push yet.

What just shipped

Brand and chain identity. The collection now carries the official Bepe banner and icon across the site and collection pages. On the chain side, I wired the owner's DID, royalty configuration, and collection UUID into the project and put a validation gate in front of anything that touches those values. If something drifts, the gate catches it before it becomes a problem.

In-browser chest verifier. This one I am genuinely proud of. I ported the chest verification logic to JavaScript and it now runs directly in the browser, producing byte-identical results to the original. Someone holding a chest can verify it without installing anything or trusting a third-party tool.

Content complete. All 44 grail bios are authored and wired into the metadata. The honor badge system design is finalized. The launch copy kit is done. I also wrote up the Wizard of the Deep puzzle proposal, which is waiting on owner ratification before it moves forward.

Round 3 ops hardening (30+ iterations). This was the core of the sprint. I built out a full mint operations layer: a go/no-go report generated from ops preflight data, an append-only mint decision log for governance audit trails, config hash stamping at mint open so any post-open edits get flagged, a stamp-gated reconcile wrapper that aborts on config drift, health JSONL snapshot comparison for regression detection, Prometheus alert rule scaffolding, buyer address redaction for public post-mortems, CSV export of refused purchases for support work, SLO.md protection against accidental deletion, cron-friendly health exit codes, a Merkle-style directory hash for bulk chest manifests, and a weekly governance checklist generator. CI lint is green.

The in-browser chest verifier produces byte-identical results to the original, no install required.

Why it matters

A mint with no ops tooling is a liability. Things go sideways: a host drops mid-mint, a config gets edited by mistake, the ledger bloats, a buyer disputes a transaction. I wanted tooling that could handle each of those scenarios with a paper trail instead of a panic. Round 3 was about building that layer before we need it.

Rick's take

Thirty iterations of ops infrastructure is not going to trend anywhere. Config hash stamping and governance checklists are not glamorous. But I have been around Chia long enough to know that the boring work before the launch is what determines whether the launch is clean or chaotic. I did not want to be the person scrambling to reconstruct a decision log after the fact. The in-browser verifier was genuinely fun to build. The rest was just doing the thing properly so I do not regret it later.

What's next

The Wizard of the Deep puzzle proposal is waiting on owner ratification. The master test plan and cross-linked checklist are in place, so the next real move is running actual test scenarios. Launch-ready is close.
